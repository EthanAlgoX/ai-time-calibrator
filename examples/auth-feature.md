# Example: Authentication Feature

## Request

Add email/password login, logout, protected routes, and basic session handling to an existing web app.

## Traditional Estimate

4 days

## AI-Adjusted Estimate

- Optimistic: 1.5 days
- Expected: 2.5 days
- Conservative: 4 days

## Why It Changed

- AI can accelerate route setup, form handling, validation, tests, and repeated framework code.
- Security-sensitive decisions, session behavior, and edge cases still need careful review.
- Existing auth patterns or libraries move the estimate toward optimistic.

## Risks

- Password reset, rate limiting, and account lockout requirements may expand scope.
- Security review can dominate implementation time.
- Legacy session behavior may create integration risk.
